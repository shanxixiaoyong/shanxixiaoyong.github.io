# Love 2048 Fate Events and 5x5 Board Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Convert Heartbeat 2048 to a mobile-first 5x5 board and add a restrained, pity-backed fate/conflict event system that accelerates late stages without interrupting swipes.

**Architecture:** Add a small UMD-style pure rules module so line movement, event directing, fate resolution, and conflict repair can be tested with Node's built-in test runner. Keep DOM rendering and milestone cinematics in `assets/games.js`, using primitive tile values (`number`, `"fate"`, `"conflict:2"`, `"conflict:1"`, `"destiny"`) to avoid a broad rewrite of unrelated games.

**Tech Stack:** Static HTML, CSS, browser JavaScript, Node.js built-in `node:test`, existing Love2048 canvas VFX.

## Global Constraints

- The only public game is named exactly `心动2048`; the other nine games, their public pages, links, registry entries, and runtime implementations are removed.
- The playable board is exactly 5 columns by 5 rows.
- Random events are disabled before the current highest tile reaches 2048.
- Only two visible special tile types exist: `fate` and `conflict`.
- Special events replace the normal top-row spawn; they never add a second spawn in the same turn.
- Blocked directional input still creates the normal top-row tile when space exists, but does not accumulate fate effort or roll a random event.
- Two fate tiles merge into a destiny resolution: upgrade one current highest ordinary tile by 2x and clear up to four lowest ordinary tiles.
- A once-per-run miracle triggers when fate resolves with zero or one empty cell: upgrade the highest tile and clear every other ordinary tile.
- The second fate tile is guaranteed no later than the fourth eligible turn after the first fate tile appears.
- A conflict tile has two cracks and disappears after two ordinary numeric merges; it never freezes, spreads, or upgrades.
- Conflict never appears with fewer than five empty cells, while fate is active, during the eight-turn milestone grace period, or while another conflict is active.
- First-time stages created by destiny use the existing full-screen milestone cinematic; repeated stages use only a short board effect.
- Event effects must not block pointer input or pause continuous swiping.
- Target viewport for visual QA is 430x932 CSS pixels, with an additional 393x852 overflow check.
