# Distinct Arcade Games Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the arcade games visibly and mechanically distinct while keeping the public page clean.

**Architecture:** Keep one static page and one game runtime, but add explicit per-game metadata, per-game DOM classes, and per-game CSS systems. Validation guards against a return to generic shared demos.

**Tech Stack:** Static HTML, CSS, vanilla JavaScript, Node validation script, Chrome headless verification.

## Global Constraints

- Frontend must not include internal requirements or implementation notes.
- No side touch-pad or global direction panel.
- Each game must have a unique theme token, feature copy, board class, and gameplay-specific state.
- Images and assets remain local.
- GitHub Pages URL remains `https://shanxixiaoyong.github.io/`.
